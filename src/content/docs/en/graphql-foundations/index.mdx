GraphQL is a **query language for APIs** paired with a **type system** that describes the data those APIs can return. Instead of calling many fixed URLs and hoping each one gives back the right shape, a client sends one request to a single endpoint and asks for *exactly* the fields it wants. The server replies with a response shaped to match the request — nothing more, nothing less.

That single idea changes a lot. The data your API can serve becomes a connected **graph** of types, and a query is simply a path you trace through that graph.

## What this module covers

This is the foundations module of the course. By the end of it you will be able to read GraphQL, explain why teams reach for it, and reason about how a server turns a query into a response. The five lessons are:

## The shape of a GraphQL system

<Mermaid title="A client asks one endpoint for exactly the fields it needs." code={overviewDiagram} />

A client builds a query describing the fields it needs. The server holds a schema (the contract) and a set of resolver functions (the logic). It checks the query against the schema, runs the resolvers to gather data, and returns a response that mirrors the query's shape.

## Your first real query

Below is a working GraphQL example. It defines a tiny schema with one field, `greeting`, provides a function to resolve it, and runs a query. Press **Run** and watch the actual GraphQL engine produce a JSON result in the output panel.

export const helloCode = `(async () => {
  const { graphql, buildSchema } = await import('https://esm.sh/graphql@16');

  // The schema is the contract: it says a Query has a "greeting" field
  // that takes an optional "name" and returns a String.
  const schema = buildSchema(\`
    type Query {
      greeting(name: String): String
    }
  \`);

  // The root holds the logic that produces each field's value.
  const root = {
    greeting: ({ name }) => 'Hello, ' + (name || 'world') + '! Welcome to GraphQL.',
  };

  // Ask for exactly one field, passing an argument.
  const query = '{ greeting(name: "learner") }';

  const result = await graphql({ schema, source: query, rootValue: root });
  console.log(JSON.stringify(result, null, 2));
})();`;

<NodeRunner client:visible code={helloCode} />

Notice the result is wrapped in a `data` key, and inside it the field name `greeting` maps to the value the resolver returned. The response is predictable because it follows the query. Try changing the name argument, or the field, and run it again.

<Callout title="The mental model to carry forward">
GraphQL is not a database and not a replacement for your business logic. It is a typed contract sitting in front of your data, letting clients describe their needs precisely. Hold onto three words — **schema**, **query**, **resolver** — and the rest of this course will click into place.
</Callout>
